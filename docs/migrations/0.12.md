# HardKAS 0.12 Migration Guide

HardKAS 0.12 represents the "Safe Pruning" phase where experimental aliases and some legacy patterns were physically removed from the public API to reduce surface area and improve maintainability.

## Breaking Changes

### Removed Experimental Aliases
The following experimental aliases have been removed as their underlying features are now fully integrated into the core product:
- `hardkas.experimental.toccata` → Use `hardkas.covenants` instead.
- `hardkas.experimental.fees` → Use `hardkas.fees` instead.
- `hardkas.experimental.igra` → Use `hardkas.l2.igra` instead.
- `HardkasToccata` interface → Use `HardkasCovenants` instead.

### Removed legacyDeploy
The `legacyDeploy()` method has been removed. Use `planDeploy()` instead, which follows the modern transaction planning pipeline.

### Internalized APIs
Several wire-level serialization, testing, and transport details have been internalized. They are no longer exported from public package barrels:
- `parseWasmTxToRpc` and related wire-level WASM/RPC translation functions.
- Raw RPC notification method constants.
- Internal transport callbacks and subscription registries from `@hardkas/kaspa-rpc`.

### Testing Signers
Some testing signers previously exported by `@hardkas/accounts` have been moved to `@hardkas/testing` to prevent polluting the production package:
- `FixtureSigner`
- `SimulatedTxPlanSigner`
- `FakeSigner`

You can import them like this:
```typescript
import { FixtureSigner } from "@hardkas/testing";
```

*Note: `KaspaWasmPrivateKeySigner`, `PrivateKeyAuthorizer`, and `StaticSignatureScriptAuthorizer` remain publicly exported in `@hardkas/accounts`.*
